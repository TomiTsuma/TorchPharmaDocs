torch_pharma.models.diffusion

3D equivariant diffusion models for de-novo molecule generation.

class torch_pharma.models.diffusion.EquivariantVariationalDiffusion(dynamics_network, dataset_info, num_atom_types=16, num_x_dims=3, include_charges=False, num_timesteps=1000, parametrization='eps', noise_schedule='polynomial_2', noise_precision=1e-5, loss_type='l2', norm_values=(1.0, 4.0, 10.0), norm_biases=(0.0, 0.0, 0.0), self_condition=True, diffusion_target='atom_types_and_coords')

Equivariant Variational Diffusion (EVD) model. Jointly diffuses 3D atom coordinates (x) and atom-type features (h) using a continuous-time forward process and a learned denoising backbone. The forward process is translation-invariant: coordinate noise is projected onto the zero centre-of-gravity subspace at every step.

Parameters

  • dynamics_network (nn.Module) – Denoising backbone (e.g. EGNNDynamics or GCPNetDynamics). Must accept a noisy batch and return predicted noise.
  • dataset_info (dict) – Must contain an 'n_nodes' key with a histogram dict {num_nodes: count} used to build the node-count prior.
  • num_atom_types (int) – Vocabulary size of the one-hot atom-type encoding. Default: 16.
  • num_x_dims (int) – Spatial dimensionality of atom coordinates. Always 3 for molecules. Default: 3.
  • include_charges (bool) – If True, integer atomic charge is appended to h and jointly diffused. Default: False.
  • num_timesteps (int) – Total diffusion steps T. Default: 1000.
  • parametrization (str) – Noise parametrization. Only 'eps' (predicting added noise) is supported. Default: 'eps'.
  • noise_schedule (str) – One of 'cosine', 'polynomial_2', 'polynomial_n', or 'learned'. 'learned' requires loss_type='vlb'. Default: 'polynomial_2'.
  • noise_precision (float) – Numerical clipping floor applied to polynomial schedule values. Default: 1e-5.
  • loss_type (str) – 'l2' for simple MSE loss or 'vlb' for variational lower bound. 'vlb' is required when noise_schedule='learned'. Default: 'l2'.
  • norm_values (tuple[float, ...]) – Per-channel scale factors applied before diffusion for [coordinates, categorical features, integer features]. Default: (1.0, 4.0, 10.0).
  • norm_biases (tuple[float, ...]) – Per-channel bias applied before diffusion. Default: (0.0, 0.0, 0.0).
  • self_condition (bool) – Enable self-conditioning: the previous prediction is fed back as an additional input with probability 0.5 during training. Default: True.
  • diffusion_target (str) – Specifies which features are jointly diffused. Currently only 'atom_types_and_coords' is supported. Default: 'atom_types_and_coords'.

Example

from torch_pharma.models.diffusion import EquivariantVariationalDiffusion
from torch_pharma.models.dynamics.egnn import EGNNDynamics

dynamics = EGNNDynamics(num_atom_types=5, h_hidden_dim=256, num_encoder_layers=9)

model = EquivariantVariationalDiffusion(
    dynamics_network=dynamics,
    dataset_info=dataset_info,   # must contain "n_nodes" histogram
    num_atom_types=5,
    noise_schedule="polynomial_2",
    num_timesteps=1000,
)

# Training step
loss, *_ = model(batch)
loss.backward()

# Sampling
from torch_pharma.utils.io import num_nodes_to_batch_index
num_nodes = torch.tensor([19] * 32)
samples = model.sample(num_samples=32, num_nodes=num_nodes)

Methods

forward(batch, return_loss_info=False)

Compute the diffusion training objective. Samples a random timestep t, computes q(z_t | x), runs the denoising network, and returns the loss.

  • batch (torch_geometric.data.Batch) – Batched molecular graph with attributes: x (coords), h (atom types), batch (graph index), node_mask.
  • return_loss_info (bool, optional) – If True, also return a dict of individual loss components. Default: False.

Return type Tuple[Tensor, ...] — (total_loss, loss_t, loss_0, loss_vlb, kl_prior)

sample(num_samples, num_nodes, context=None, fix_noise=False)

Ancestral sampling. Runs the reverse diffusion chain from t=T to t=0 and decodes into atom coordinates and types.

  • num_samples (int) – Number of molecules to generate.
  • num_nodes (Tensor) – 1D long tensor of shape [num_samples] specifying how many atoms each generated molecule should have.
  • context (Tensor, optional) – Property conditioning tensor of shape [num_samples, num_context_features]. Default: None.
  • fix_noise (bool, optional) – If True, use the same noise realisation across all sampling steps for visualisation. Default: False.

Return type Tuple[Tensor, Tensor] — (x, h) atom coordinates and one-hot atom types

compute_noised_representation(xh, batch_index, node_mask, gamma_t)

Compute z_t = α_t · xh + σ_t · ε given a pre-sampled noise level gamma_t.

Return type Tuple[Tensor, Tensor] — (z_t, epsilon)

compute_kl_prior(xh, batch_index, node_mask, num_nodes, device)

Analytical KL divergence KL[q(z₁|x) ∥ p(z₁)] between the fully-noised distribution at t=T and the unit Gaussian prior. Computed separately for coordinates and scalar features then summed.

Return type Tensor — per-graph KL values, shape [batch_size]

normalize(x, h, node_mask)

Apply norm_values and norm_biases to scale x and h before diffusion.

Return type Tuple[Tensor, dict] — (x_norm, h_norm_dict)

unnormalize(x, h_cat, h_int, node_mask)

Inverse of normalize. Recovers original-scale features from normalized form.

Return type Tuple[Tensor, Tensor, Tensor] — (x, h_cat, h_int)

class torch_pharma.models.diffusion.PredefinedNoiseSchedule(noise_schedule, timesteps, precision)

A non-learnable noise schedule. Precomputes a buffer of γ values from a chosen analytic curve indexed by integer timestep. At forward time performs a rounded integer lookup — O(1) and gradient-free.

Parameters

  • noise_schedule (str) – Schedule type: 'cosine', 'polynomial_2', or 'polynomial_n' where n is an integer exponent.
  • timesteps (int) – Total number of diffusion steps T. Buffer length = T + 1.
  • precision (float) – Minimum clamp value for polynomial schedules to avoid numerical underflow.

Example

from torch_pharma.models.diffusion.noise import PredefinedNoiseSchedule

sched = PredefinedNoiseSchedule(
    noise_schedule="polynomial_2",
    timesteps=1000,
    precision=1e-5,
)
t = torch.tensor([0.5])
gamma = sched(t)  # → scalar log-SNR

Methods

forward(t)

Look up the precomputed γ for a normalized time t ∈ [0, 1]. Converts to integer index via rounding.

  • t (Tensor) – Normalized timestep tensor, values in [0, 1].

Return type Tensor — γ(t), same shape as t

class torch_pharma.models.diffusion.GammaNetwork()

A learnable monotone noise schedule. Implemented as a small MLP built from PositiveLinear layers, ensuring γ is strictly increasing in t. Valid only when loss_type='vlb'.

Note: GammaNetwork learns γ₀ (≈ -5) and γ₁ (≈ 10) as trainable scalar parameters. The output is renormalized so that γ(0) = γ₀ and γ(1) = γ₁ at all times.

Example

from torch_pharma.models.diffusion.noise import GammaNetwork

sched = GammaNetwork()
t = torch.linspace(0, 1, 100)
gamma = sched(t)   # learned γ(t) curve

Methods

forward(t)

Evaluate the learned schedule at normalized times t ∈ [0, 1].

  • t (Tensor) – Normalized timestep tensor.

Return type Tensor — γ(t)

class torch_pharma.models.diffusion.PositiveLinear(in_features, out_features, bias=True)

A linear layer whose weights are constrained positive by applying softplus. Used as the building block of GammaNetwork to guarantee a monotonically increasing noise schedule.

Parameters

  • in_features (int) – Size of each input sample.
  • out_features (int) – Size of each output sample.
  • bias (bool) – If True, adds a learnable bias. Default: True.
