class torch_pharma.data.datasets.qm9.QM9Dataset(calculate_thermo=True)

The QM9 quantum chemistry benchmark. Contains ~134k organic molecules (up to 9 heavy atoms from H, C, N, O, F) annotated with 19 electronic, energetic, and geometric properties computed by DFT. Inherits from BaseDataset.

Parameters

  • calculate_thermo (bool) – If True, subtract per-atom thermochemical energy offsets from the raw DFT targets during processing. Default: True.

Example

from torch_pharma.data.datasets.qm9 import QM9Dataset

ds = QM9Dataset(calculate_thermo=True)
ds.process()    # download → split → parse → save .npz

Methods

download()

Fetch dsgdb9nsd.xyz.tar.bz2, uncharacterized.txt, and atomref.txt from the GDB9 figshare into TORCH_PHARMA_HOME/qm9/.

Return type None

gen_splits_gdb9(cleanup=True)

Generate reproducible train/valid/test splits. Excludes 3,054 uncharacterized molecules. Assigns 100k to training, ~13k to test (10%), remaining to validation.

Return type dict — {'train': ndarray, 'valid': ndarray, 'test': ndarray} of molecule indices

get_thermo_dict()

Parse atomref.txt and return per-element thermochemical energies for targets: zpve, U0, U, H, G, Cv.

Return type dict[str, dict[int, float]] — {target: {atomic_charge: energy}}

add_thermo_targets(data, therm_energy_dict)

Subtract per-atom thermochemical energies from raw DFT values and add new <target>_thermo keys to the split dict.

  • data (dict) – Dataset split dictionary with at least 'charges' and one target array per thermochemical target.
  • therm_energy_dict (dict) – Output of get_thermo_dict().

Return type dict — same structure as data with additional _thermo keys

get_unique_charges(charges)

Count the multiplicity of each atomic charge across all molecules.

Return type dict[int, ndarray] — {charge: count_per_molecule}

process()

Full pipeline: download → gen_splits_gdb9 → process_xyz_files → optional add_thermo_targets → save compressed .npz files.

Return type None

compute_smiles(dataset, remove_h)

Iterate the training loader, build RDKit molecules via build_molecule(), and return SMILES strings via mol2smiles().

Return type list[str] — SMILES strings for successfully converted training molecules

class torch_pharma.data.datasets.base.BaseDataset()

Abstract base class for all torch_pharma datasets. Subclasses must implement download() and process(). The class provides a standard lifecycle: download raw files → process to tensors → retrieve via __getitem__.

Methods

download()

Fetch raw data files from remote sources. Must be implemented by subclasses.

Return type None

process()

Transform raw files into model-ready tensors and persist to disk. Must be implemented by subclasses.

Return type None

__getitem__(idx)

Return a single data sample by index. Must be implemented by subclasses.

Return type Any

__len__()

Return the number of samples in the dataset.

Return type int

torch_pharma.data.datasets.utils.get_dataset_info(dataset_name, remove_h)

Return a dataset_info dictionary used widely across the framework for encoding, decoding, visualizing, and evaluating molecules.

Parameters

  • dataset_name (str) – Name of the dataset, e.g. 'qm9' or 'qm9_second_half'.
  • remove_h (bool) – If True, return a version of the info dict that excludes hydrogen atoms.

Return type

dict with keys: atom_encoder (dict[str, int]) — element symbol → one-hot index atom_decoder (list[str]) — one-hot index → element symbol n_nodes (dict[int, int]) — histogram {num_atoms: count} for NumNodesDistribution colors_dic (dict[str, str]) — element → hex color for visualization radius_dic (dict[str, float])— element → atomic radius for visualization

Example

from torch_pharma.data.datasets.utils import get_dataset_info

info = get_dataset_info("qm9", remove_h=False)
print(info["atom_decoder"])  # ['H', 'C', 'N', 'O', 'F']
torch_pharma.data.datasets.utils.process_xyz_files(data, process_file_fn, file_idx_list=None, stack=False)

Extract .xyz files from a .tar.bz2 archive and process each with a user-supplied function. Optionally filters to a specific list of file indices.

Parameters

  • data (str or Path) – Path to the .tar.bz2 archive.
  • process_file_fn (callable) – Function applied to each extracted file. E.g. process_xyz_gdb9.
  • file_idx_list (ndarray, optional) – Integer indices of files to extract. If None, all files are processed. Default: None.
  • stack (bool) – If True, stack the returned arrays along axis 0. Default: False.

Return type

dict[str, ndarray] — keys are property names, values are stacked arrays

torch_pharma.data.datasets.utils.process_xyz_gdb9(datafile)

Parse a single GDB9-format XYZ file. Reads atom symbols, positions, and all 19 molecular properties from the standard GDB9 file format.

Parameters

  • datafile (file-like) – Open file object pointing to a valid GDB9 .xyz file.

Return type

dict[str, ndarray] — keys: 'positions', 'charges', 'num_atoms', plus one key per molecular property (e.g. 'mu', 'alpha', 'homo', ...)
